Main menubar: New HTML File tests

New HTML File: Folder ("Where") controls

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. From the New menu on the banner, choose "HTML File..." and make sure the Composition type is set to "Desktop hi-fi page".
    First time usage, there should be a "Where" drop-down button that says "(root)". Clicking on the drop-down button should show a menu with the single value of "(root)".
  3. Click on the right-pointing arrow next to the drop down button.
    You should see a tree control that shows contents of cloud workspace. Under the tree control there should be a New Folder button. The small right-pointing arrow should have changed to an upward-pointing arrow.
    Click on "New Folder" button and create a new folder (e.g., folder1).
    The folder you created should appear in the Tree control and become the new current "where" folder.
    Clicking on the "Where" drop-down button should show a menu with the values "folder1" and "(root)".
  4. Click "Create" to create a new HTML file
  5. Drag a Button onto the new HTML file
  6. Save and close. Reopen by double-clicking on the new HTML file in the Files palette.

Expected results

  1. The Files palette should show the new HTML file in the folder you just created.
  2. The Button should look correct (verifying the logic that manages relative references to JS and CSS files within the new HTML file).

Desktop composition types

Steps

  1. From the New menu on the banner, choose "HTML File..."
  2. Create an HTML file with each possible desktop composition type ("Desktop hi-fi page", "Desktop hi-fi sketch", ...), and drag/drop a Button into the middle of each of each of these pages.

Expected results

  1. The composition type should be sticky - if your previous page was "Desktop hi-fi sketch" then the subsequent New HTML File dialog should show that as current choice.
  2. Desktop hi-fi page should open with Claro theme and flow layout (i.e., Button gets shoved to top/left of page).
  3. Desktop hi-fi sketch should open with Claro theme and absolute layout (i.e., Button gets be placed at drop location).
  4. Desktop lo-fi page should open with Sketch theme and flow layout (i.e., Button gets shoved to top/left of page).
  5. Desktop lo-fi page should open with Sketch theme and absolute layout (i.e., Button gets be placed at drop location).

Mobile composition types

Steps

  1. From the New menu on the banner, choose "HTML File..."
  2. Create an HTML file with each possible mobile device, and from dojox.mobile widget set, drag/drop a View and Heading into the middle of each of each of these pages.

Expected results

  1. The composition type should be sticky - if your previous page was "blackberry" then the subsequent New HTML File dialog should show that as current choice.
  2. The correct mobile device silhouette and correct mobile theme should be used for each device.

Theme button - custom desktop theme

Steps

  1. Create a new desktop theme (e.g., claro2) using the New Theme command.
  2. Change global background color to something else (e.g., pink).
  3. Click on Save button.
  4. New HTML File..., choose composition type of "Desktop hi-fi page".
  5. Click on "Theme..." button. In dialog that comes up, set desktop theme to "claro2" (or whatever the new theme's name is).
  6. Back in New HTML File dialog, click Create.
  7. Add a Calendar and Button.

Expected results

  1. Calendar and Button should be styled with claro2 (e.g., with pink background).

Theme button - custom mobile theme

Steps

  1. Create a new mobile theme (e.g., custom2) using the New Theme command and cloning the "custom" theme.
  2. Change global background color to something else (e.g., pink).
  3. Click on Save button.
  4. New HTML File..., choose composition type of "iphone".
  5. Click on "Theme..." button. In dialog that comes up, change mobile theme from "(device specific)" to "custom2" (or whatever the new theme's name is).
  6. Back in New HTML File dialog, click Create.
  7. Add a View and a Heading.

Expected results

  1. The View and Heading should reflect the styling of the custom2 theme.
