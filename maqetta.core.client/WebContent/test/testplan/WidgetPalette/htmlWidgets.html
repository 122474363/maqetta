Widget Palette: HTML Element Tests

Audio, Video, Embed

Steps

  1. Choose New HTML File from the New menu
  2. Open the HTML folder in the Widget palette.
  3. Drag an <audio> widget onto the canvas.
  4. Notice Smart Input dialog to select a source is displayed. Also, note there should not be a field for alt text (like there is in the Smart Input for <img>'s) because <audio> doesn't support alt text.
  5. Enter the file name for a non-existent file (e.g. this_is_a_bad_filename) and click OK. Ensure the <audio> element still has a size and is available for selection. (In Chrome, there will be partial playback controls. And, in FF, there will just be an empty box.)
  6. Double-click the <audio> element to bring Smart Input back up and specify a source:
    • If you have an audio file uploaded to your workspace, you may navigate to it and select it.
    • If you do not have your own audio file, you can enter one of the URL's below (or any other URL pointing to an audio file):
      • http://www.w3schools.com/html5/song.mp3
      • http://www.w3schools.com/html5/song.ogg
      • (NOTE: None of the major browsers support all of the various audio formats (see Audio Formats and Browser Support). For example, the mp3 file above will work in Chrome but not Firefox. So, if doing Firefox testing you'd probably want to pick the ogg file.)
  7. Click on Preview icon in the toolbar. This should open a new browser window showing your audio element, and you should hear sound when you click play.
  8. EXTRA CREDIT: Drop a <video> element onto the page and use one of your own files (uploaded to your workspace) or one of the following URLs:
    • http://www.w3schools.com/html5/movie.mp4
    • http://www.w3schools.com/html5/movie.ogg
  9. DOUBLE EXTRA CREDIT: Drop an <embed> element onto the page and use one of your own files (uploaded to your workspace) or the following URL:
    • http://www.w3schools.com/html5/helloworld.swf
  10. Click on the "X" close button to close the html file (without saving).

Expected results

  1. Smart Input is displayed when dropping <audio> onto the canvas. And, the dialog should not have a field for alt text.
  2. The <audio> element maintains a size even when entering the name of a missing file.
  3. When previewing the final page, you can click play and hear a sound.
