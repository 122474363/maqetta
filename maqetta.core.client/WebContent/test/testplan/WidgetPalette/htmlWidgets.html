Widget Palette: HTML Element Tests

Media (<audio>, <img>, <video>, and <embed>)

Steps

  1. Choose New HTML File from the New menu
  2. Open the HTML folder in the Widget palette.
  3. Drag an <audio> widget onto the canvas.
  4. Notice a Smart Input dialog to select a source is displayed. Also, note there should not be a field for alt text because <audio> doesn't support alt text.
  5. Enter the file name for a non-existent file (e.g. this_is_a_bad_filename) and click OK. Ensure the <audio> element still has a size and is available for selection. (In Chrome, there will be partial playback controls. And, in FF, there will just be an empty box.)
  6. Double-click the <audio> element to bring Smart Input back up and specify a source:
    • If you have an audio file uploaded to your workspace, you may navigate to it and select it.
    • If you do not have your own audio file, you can enter one of the URL's below (or any other URL pointing to an audio file):
      • http://www.w3schools.com/html5/song.mp3
      • http://www.w3schools.com/html5/song.ogg
      • (NOTE: None of the major browsers support all of the various audio formats (see Audio Formats and Browser Support). For example, the mp3 file above will work in Chrome but not Firefox. So, if doing Firefox testing you'd probably want to pick the ogg file.)
  7. Drag an <img> widget onto the canvas.
  8. Notice a Smart Input dialog to select a source is displayed. Also, note that unlike the dialog for <audio> there should be a field for alt text.
  9. Using the file chooser navigate to an image file (like sample_data/images/i-icon-1.png) and enter some alt text (like "my alt text"). Click OK.
  10. Click on Preview icon in the toolbar. This should open a new browser window showing your audio element, and you should hear sound when you click play. In addition, you should see your image.
  11. EXTRA CREDIT: Drop a <video> element onto the page and use one of your own files (uploaded to your workspace) or one of the following URLs:
    • http://www.w3schools.com/html5/movie.mp4
    • http://www.w3schools.com/html5/movie.ogg
  12. DOUBLE EXTRA CREDIT: Drop an <embed> element onto the page and use one of your own files (uploaded to your workspace) or the following URL:
    • http://www.w3schools.com/html5/helloworld.swf
  13. Click on the "X" close button to close the html file (without saving).

Expected results

  1. Smart Input is displayed when dropping any of the media-related tags (e.g., <audio>, <img>, <video>, and <embed>) onto the canvas. The dialog for <img> should have a field for alt text, but the dialog for <audio>/<video>/<embed> should not have a field for alt text.
  2. The <audio> element maintains a size even when entering the name of a missing file.
  3. When previewing the final page, you can click play and hear a sound. In addition, you can see your image.

<select>

Steps

  1. Create new HTML file.
  2. Drag a <select> onto the canvas.
  3. Smart Input should be displayed with 3 items. Click OK, and a drop down button should appear on the canvas.
  4. Go to widget specific properties and check the multiple attribute. The drop down should become a select list.
  5. Still within widget specific properties, enter 15 into the size field. This will cause the select field to increase in vertical size to accommodate 15 rows of data without scrolling.

<fieldset> and <input>

Steps

  1. Choose New HTML File from the New menu
  2. Open the HTML folder in the Widget palette.
  3. Drag a <fieldset> widget onto the canvas.
  4. Notice a Smart Input dialog to enter legend text is displayed. Change the value from "Legend" to one of your choosing (say "My Fieldset Legend") and click OK.
  5. Resize the <fieldset> so it has a width of roughly 400px and a height of roughly 200px.
  6. Drag an <input> onto the canvas inside of the <fieldset>. You should see a text field rendered inside of the <fieldset>.
  7. Under widget-specific properties, put a check next to disabled and change the value field to something like "my disabled input". You should see your text in a disabled text field.
  8. Drag another <input> onto the canvas inside of the <fieldset>. You should see another text field next to the first one.
  9. With the 2nd <input> selected, choose "file" from the type pull-down list under widget specific properties. The <input> that was a text field should turn into a file chooser.
  10. Click on Preview icon in the toolbar. This should open a new browser window showing your fieldset (with your caption) containing two inputs. The first input should be a disabled text field. The second input should be a file chooser (click the "Choose File"" button to confirm it brings up a file dialog).
  11. Click on the "X" close button to close the html file (without saving).

Expected results

  1. See above.
